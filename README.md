# CMX - Maintenance Evolution
Connected Maintenance 360 (CMX) 
A new product all Maintenance organizations designed to make Maintenance self‑sufficient by unifying the workflows that keep equipment moving: 
  - Inspections/defects,
  - Fault signals,
  - Work Orders,
  - Parts,
  - Vendor Work,
  - Invoices,
  - Warranty, and
  - Performance Insights

Creating a single connected experience, CMX reduces downtime and manual effort by turning operational signals into maintenance actions quickly (work orders created from inspections/defects, consolidated repair planning, automation to reduce paperwork), while ensuring outcomes can still land cleanly in the systems-of-record needed for enterprise maintenance operations.

CMX is intentionally aligned to the “connected maintenance” model seen in platforms like Samsara’s Maintenance Reimagined—where work orders can be created quickly from inspections, outstanding maintenance items are surfaced to consolidate shop visits, and invoice handling is streamlined through automation—so Maintenance teams can operate faster with less swivel-chair work.

 - 1) Self-sufficient maintenance execution (Day-to-day speed)
 
         CMX focuses on enabling Maintenance to run end-to-end workflows with a unified view of issues / history / next actions, matching the “single dashboard” approach and workflow consolidated experience.

 - 2) Reduced downtime + higher throughput (Operational impact)

        CMX standardizes how maintenance work is created, prioritized, and completed so assets spend less time in the shop and more time on the job, echoing the “increase asset uptime” value proposition of connected maintenance platforms.

 - 3) Faster, more accurate financial linkage (Invoices → Work Orders)

        CMX treats invoices and supporting documents as first-class inputs that become structured work-order outcomes that are consistent with the reality that maintenance modernization includes invoice processing and work order integration patterns (ex. extracting invoice values and             applying them to work orders).

# The GAP
Maintenance teams lose time and accuracy when operational signals (defects, faults, inspections, vendor status) and financial artifacts (invoices) do not translate cleanly into actionable, standardized work orders and outcomes. Connected platforms solve this by turning inspections into work orders quickly and consolidating outstanding maintenance items into fewer visits, while also using automation to reduce invoice and paperwork processing.
Inside J.B. Hunt, we also have enterprise realities where invoices and extracted values must integrate into work order management with defined field behaviors and exception handling (ex. rules for invoice number detection, tax handling, and how costs roll into work orders). 
CMX closes the gap by making that “signal → work → cost → outcome” path a single product experience.

# Primary Users/Personas
- Maintenance Technicians & Shop Leads
CMX reduces “paperwork time” by accelerating work order creation from inspection/defect inputs and supporting more guided execution.
- Fleet Support / Road Service / Vendor Coordination
CMX supports vendor-facing workflows like capturing equipment status and processing vendor invoices as part of an integrated maintenance loop
 - Maintenance Ops Leaders
CMX provides a consolidated operational picture that supports cost and uptime decisions, consistent with the “track costs and identify issues” and unified reporting themes from connected maintenance platforms.

# Success Metrics
Maintenance (productivity, outsource cost accuracy, warranty/claims success, predictive analytics)
 - Time from defect/fault → work order created
 - % of maintenance items consolidated into a single shop visit
 - Invoice processing cycle time (vendor invoice → structured cost posted)
 - Asset downtime reduction / throughput improvement

# AI's Role
Embedded “operational AI.” CMX bakes AI into maintenance workflows: ex. an AI assistant that suggests consolidation of repairs, auto-extracts invoice data into work orders, or even voices-out to capture vendor updates. The Maintenance Advisor is there to take actions or make decisions easier in real-time (agentic) – not just analysis. This approach stems from strategy to infuse AI in day-to-day ops (Maintenance Advisor concepts like voice-guided status updates and automatic warranty checks).
